## 2. Channel Data Model & Storage

### Stream URL Model (data/models/stream_url.dart)
```dart
@JsonSerializable()
class StreamUrl {
```

### Channel Model (data/models/channel.dart)
```dart
@JsonSerializable()
class Channel {
  final String id;
  final String name;
  final String icon; // Tabler icon name (e.g., "tabler_news")
```

## 10. M3U Import Workflow

### Import Screen (Settings > Import)
- URL input field (default: https://iptv-org.github.io/iptv/countries/ma.m3u)
- "Preview" button → fetch and parse, show summary
- Preview dialog:
  - Number of channels found
  - Conflicts (existing channels with same name)
  - Resolution options: skip/overwrite/merge
- "Import" button → executes merge
- Progress indicator
- Result: success/failure counts, error log

### Import Behavior
- **Append mode**: Add new channels, update existing URLs if missing
- **Replace mode**: Clear user channels, import fresh
- Conflict resolution:
  - Name match → ask: replace URL, keep both (suffix), skip
  - Default: append new URLs to existing channel's streamUrls list

### Post-Import
- Channel list automatically refreshed
- New channels appended with highest order numbers
- User can then reorder manually

## 13. Channel Reordering Implementation Details

### Order Persistence Strategy
```dart
// User modifications saved to: ${appDocumentsDir}/channels_user.json
{
  "version": "1.0.0",
  "lastUpdated": "2025-04-24T10:30:00Z",
  "channelOrder": ["al_aoula", "2m", "medi1", ...], // custom order IDs
  "channelOverrides": [
    {"id": "2m", "isActive": false, "streamUrls": [...]}, // partial updates
  ],
  "importedChannels": [ ... ] // channels from M3U imports
}
```

### Repository Merge Logic (data/repositories/channel_repository.dart)
1. Load bundled channels from `assets/channels.json` (read-only, default order)
2. Load user file from app documents: `channels_user.json` (if exists)
3. Apply custom order from user file's `channelOrder` array
4. Apply overrides from user file's `channelOverrides` (isActive, streamUrls)
5. Append imported channels from user file's `importedChannels`
6. Merge algorithm:
   - Channels in `channelOrder` appear in that sequence
   - Channels not in override list follow bundled order
   - Imported channels appended after bundled ones with incremental order

### Save Order Flow
1. User reorders in Settings (drag or button-based)
2. Press "Save Order"
3. Repository reads current full channel list
4. Creates `ChannelOrder` object with ordered IDs
5. Writes to `channels_user.json` in app documents directory
6. Notifies providers → TV screen updates immediately
7. File persists across app restarts and TV power cycles

## 14. M3U Import Implementation Details

### M3U Format Parsing
```
#EXTM3U
#EXTINF:-1 tvg-id="al-aoula.hd" tvg-name="Al Aoula HD" tvg-logo="https://...",Al Aoula HD
https://stream.url/...
```

Parse attributes:
- `tvg-name` → channel name
- `tvg-logo` → logo URL (optional, stored separately)
- Stream URL from next line

### Channel Matching
- Match by name (case-insensitive, normalized)
- Match by tvg-id if available
- Match by URL similarity
- User confirmation for ambiguous matches

### Import Result Reporting
```dart
class ImportResult {
  final int channelsAdded;
  final int channelsUpdated;
  final int conflictsResolved;
  final List<String> errors;
  final Set<String> duplicateNames;
}
```
